Getting Sources

Frhed is always released with sources. Sources are not (anymore) in the same binary package. Sources are in own package, named Frhed-version-src.zip. Links to latest releases are in Frhed homepage. Direct link to SourceForge.net downloads: Frhed downloads.

The latest codes are available from Mercurial repository at https://bitbucket.org/kimmov/frhed. You can either clone the repository using Mercurial software or download the source archive.

Environment

Supported Compilers

  • Visual Studio 2005 (Service Pack 1 recommended)
  • Visual Studio 2008 (Service Pack 1 recommended) - Used for releases
  • Visual Studio C++ 2010
  • Visual Studio C++ 2010 Express Edition

Visual Studio c++ 2010 Express Edition is a free compiler available from http://www.microsoft.com/Express/. You don't need to buy commercial Visual Studio edition for compiling Frhed. Express Edition does not countain resource editor but you can use any of the freely available resource editors. One good free resource editor is ResEdit.

Needed libraries

Platform SDK (PSDK) or Windows SDK is needed for compiling. Platform SDK Windows 2003 SP1 or later is recommended. Download and install.

Frhed uses htmlhelp library to show HTML help files in help viewer. The HtmlHelp library is available as a part of HTML Help Workshop and as part of Platform SDK (PSDK).

Project Files

The sources contain solution and project files for Visual Studio 2008. But later Visual Studio versions convert those files to new format.

Frhed consist of three projects:

  • Frhed - the main project
  • heksedit - the editor window project (embeddable into other apps)
  • rawio32 - I/O utility project
The dependencies are such that:
  • Frhed project uses heksedit editor component.
  • heksedit project uses rawio32 project.

Compile

To compile Frhed:

  • Open FRHED.sln solution file to Visual Studio
  • Select the frhed project as a startup project
  • Select Rebuild all to compile
